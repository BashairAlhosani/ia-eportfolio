Team Project: Intelligent Academic Research Agent System

Team Project Multi-Agent System Python Development AOSE Paradigm

Project Overview

Developed an intelligent agent-based system utilizing Python to automate academic research discovery through multi-agent architecture. The system employed four specialized agents (Query Agent, Retrieval Agent, Processing Agent, Storage Agent) working collaboratively to search, retrieve, process, and store academic research findings from online platforms.

Key Features:

  • Multi-Agent Architecture: Four specialized agents with distinct responsibilities coordinated through KQML performatives
  • Hybrid BDI Model: Combined deliberative planning with reactive responsiveness for optimal performance
  • Dynamic Content Handling: Utilized Selenium for JavaScript-heavy websites and Requests for static content
  • Intelligent Processing: Applied filtering and relevance ranking algorithms for quality assurance
  • Offline Storage: Persistent data storage in JSON/CSV formats for future analysis

My Contribution: Member 3 - Challenges & Paradigms Analysis

"As Member 3, I conducted comprehensive analysis of implementation challenges and paradigm selections, examining critical factors affecting system performance, reliability, and scalability while justifying architectural decisions through established agent communication theories."

Challenge Areas Analyzed:

1. Data Quality and Retrieval Challenges

Problem: Improper search results from dynamic websites and general queries affecting result accuracy and relevance.

Solution Proposed: Implemented adaptive filtering strategies using keyword refinement algorithms and source credibility scoring. Recommended integration of multiple data sources and cross-validation mechanisms to ensure result quality.

2. Agent Workload and System Performance Issues

Problem: Risk of overloaded agents when Retrieval Agent makes multiple requests to busy academic platforms, potentially causing bottlenecks and system degradation.

Solution Proposed: Justified hybrid agent paradigm approach combining reactive responsiveness with deliberative planning. Recommended asynchronous processing patterns and load balancing strategies to distribute workload efficiently across agents.

3. Interpretation, Relevance, and Semantic Alignment Problems

Problem: Relevancy filtering errors where Processing Agent might misclassify results or accidentally exclude useful academic information.

Solution Proposed: Applied rational intention revision mechanisms based on BDI model principles (Bratman et al., 1988). Recommended adaptive learning algorithms to improve classification accuracy over time through continuous feedback loops.

Theoretical Justification:

  • Agent Communication: Grounded approach in Searle's (1969) speech act theory operationalized through KQML performatives (Finin et al., 1994)
  • BDI Architecture: Leveraged Bratman et al.'s (1988) Belief-Desire-Intention model for autonomous decision-making
  • Hybrid Paradigm: Balanced computational efficiency with strategic reasoning through combined reactive and deliberative approaches

Individual Project: Intelligent Research Assistant

Individual Project API Integration Data Processing Python

Project Overview

Developed an automated system to help researchers efficiently discover, clean, and store academic research papers. The Intelligent Research Assistant addresses the challenge of manual literature review by creating a streamlined pipeline that retrieves scholarly metadata from Semantic Scholar API, applies systematic data cleaning algorithms, and stores results in structured CSV format for easy analysis and sharing.

Project Goals:

  • Automation: Eliminate manual searching through automated retrieval of academic articles with valid scholarly metadata including titles, authors, abstracts, publication years, and URLs
  • Data Quality: Implement data cleaning and filtering methods to remove low-quality or incomplete entries, duplicates, and irrelevant publications
  • Structured Storage: Store cleaned data in organized CSV format for easy inspection, sharing, and analysis through spreadsheets or programming pipelines
  • Transparency: Provide detailed logging at every key step to aid debugging, verification, and academic demonstration
  • Reliability: Ensure consistent, efficient, and replicable research discovery process

System Architecture & Design

The Intelligent Research Assistant follows a modular architecture emphasizing separation of concerns, maintainability, and scalability. Each component performs specialized tasks with well-defined data flows between modules.

1. QueryBuilder Module - Input Validation

Function: Receives user input for research topics and validates to ensure quality before downstream processing.

Implementation: Enforces input correctness at system boundary by rejecting invalid, empty, or malformed inputs. Normalizes valid inputs and converts them into structured query format compatible with Semantic Scholar API.

Significance: Prevents unnecessary API failures and ensures downstream modules receive valid input, demonstrating sound software engineering principles.

2. SemanticScholarRetriever Module - API Integration

Function: Interfaces with Semantic Scholar API to retrieve academic papers based on validated queries.

Implementation: Processes HTTP requests requesting specific fields (title, authors, abstract, year, URL). Implements retry logic to handle network errors and API rate limiting, ensuring robustness during demonstrations and batch queries.

Data Source: Semantic Scholar selected for free, high-quality academic metadata access without authentication requirements, making it ideal for academic projects.

3. DataCleaner Module - Quality Assurance

Function: Processes raw retrieved data by implementing quality control procedures.

Implementation:

  • Removes papers lacking abstracts (abstracts provide critical relevance assessment)
  • Eliminates duplicate entries by comparing paper titles
  • Applies optional filtering by publication year for temporal analysis

Output: Concise, relevant, academically-usable dataset free from redundancy and low-quality entries.

4. DataStorage Module - Output Management

Function: Stores processed results in structured, accessible format.

Implementation: Saves data in CSV (Comma-Separated Values) format - human-readable, broadly supported, and easily imported into Excel, Python, R, and visualization software. Automatically creates output directory if non-existent, organizes files by research topic, and displays preview of initial rows in terminal for immediate verification.

Benefits: Maximizes usability and accessibility for analysis, sharing, and presentation purposes.

Implementation Highlights

Key Technical Features:

Modular Architecture

System organized into four specialized modules (QueryBuilder, SemanticScholarRetriever, DataCleaner, DataStorage) with clear separation of concerns. Each module independently testable and maintainable, facilitating future enhancements and debugging.

Robust Error Handling

Implemented retry mechanisms for API requests to handle network failures and rate limiting. Input validation prevents downstream errors. Comprehensive error logging provides transparency and facilitates troubleshooting.

Data Quality Controls

Three-tier filtering system: (1) Abstract presence verification ensuring meaningful content, (2) Duplicate removal based on title comparison maintaining dataset integrity, (3) Optional year-based filtering enabling temporal trend analysis. These controls guarantee final dataset is concise, relevant, and academically valuable.

Comprehensive Logging

Detailed informational messages at every processing step provide visibility into system execution. Logs display number of papers fetched, cleaning operations performed, and storage confirmations. Essential for debugging, verification, academic demonstrations, and assessment evidence collection.

Structured Output

CSV format selected for universal compatibility and ease of use. Automatic directory creation and organized file naming by research topic ensures logical arrangement and traceability. Terminal preview feature enables immediate output verification without file opening.

Testing & Verification

Comprehensive unit testing implemented using Python's unittest framework to guarantee system reliability and accuracy. Testing validates individual modules and demonstrates robustness under real-world conditions.

Test Coverage:

QueryBuilder Module Tests

Validates proper input checking and query construction. Ensures invalid inputs are rejected, preventing downstream errors and system failures.

DataCleaner Module Tests

Tested with real retrieved papers to verify duplicate removal and incomplete record filtering. Real-world testing enhances confidence in practical applicability and data quality assurance.

DataStorage Module Tests

Verifies CSV file generation, correct directory storage, and accurate cleaned data persistence. Ensures output integrity and accessibility.

Test Results:

Status: All unit tests passed successfully

Significance: Terminal output showing successful test execution provides concrete evidence of system functionality, reliability, and academic credibility. Testing strategy ensures maintainability for future enhancements.

System Demonstration & Performance

End-to-end system demonstration provides practical evidence of accurate implementation and operational effectiveness:

Step 1: User Input

Program prompts user for research topic in terminal. Input undergoes validation and structured query creation.

Step 2: API Retrieval

Semantic Scholar API searches and retrieves papers matching topic. Informational log messages display number of papers fetched and processing status.

Step 3: Data Cleaning

System applies filtering (duplicate removal, abstract-based filtering). Logs notify number of papers remaining after cleaning operations.

Step 4: Storage & Preview

Cleaned data saved to output directory as CSV file. Terminal displays preview showing first few papers with titles, authors, publication year, abstracts, and URLs.

Step 5: Verification

Unit test execution results displayed showing all tests successful. Complete workflow provides assessment evidence and demonstrates end-to-end functionality.
